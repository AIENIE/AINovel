# G1 引导创作

## 当前边界

G1-A P0 覆盖 `/novels/quick-create` 的 Step0-4：一句话起点、故事方向、世界设定、角色阵容和章节大纲。传统新建小说与现有 `/v1/conception` 保持兼容。

- 故事、世界和角色步骤返回恰好 3 个候选；大纲先返回 3 条简要根方向。
- 角色候选每套包含 3-5 名角色。
- 三条大纲方向各自保留一份最新发展稿，可继续发展、按反馈重写并随时切换；选定后才展开一套完整大纲。
- 完整大纲严格匹配 3-12 章目标，每章包含 2-4 个场景，预览可编辑后才确认落库。
- 世界设定允许跳过；用户可以修改候选字段后确认。
- 自动模式采用推荐候选并在后台继续，草稿可跨设备恢复。

分章正文草稿属于 G1-A P1，前置条件和验收门槛见 [`../roadmap.md`](../roadmap.md)。

## 状态与持久化

- `creation_workflow_runs` 保存用户、当前步骤、确认上下文、标准实体引用、自动模式和乐观版本号。
- `async_jobs` 对普通步骤使用运行/步骤幂等键，对方向发展、重写和展开使用运行/方向/修订号/操作幂等键，记录进度、尝试次数、用量、错误与租约。
- 同一工作流始终只有一个活跃 AI 任务；大纲步骤在 `DIRECTION_SELECTION` 与 `OUTLINE_PREVIEW` 两个阶段之间推进。
- 任务状态为 `QUEUED -> RUNNING -> CALLING_AI -> SUCCEEDED/FAILED`。
- 服务重启时，尚未调用 AI 的任务重新排队；停在 `CALLING_AI` 的任务转为 `RECOVERY_REQUIRED`，必须由用户明确重试。
- 周期协调器补偿自动模式中的中间状态，不静默重复不确定的 AI 调用。

## 标准实体落库

用户确认后在事务内写入现有模型：故事方向写入 `Story`，世界设定写入 `World`，角色阵容写入 `CharacterCard`，章节大纲写入 `Outline`。向导不维护第二套故事数据模型。

## AI 与计费

- 普通候选、简要方向、方向发展/重写和完整展开使用独立提示词版本。
- AI 调用通过现有 `AiService` 和实际用量计费，账本引用类型为 `G1_WORKFLOW`。
- 同一任务重试复用计费幂等引用。
- 大纲步骤累计记录多次方向操作用量，单个任务只记录本次调用用量。
- 首次 JSON 结构不合格时最多追加一次结构修复调用；仍不合格则任务失败并保留可诊断错误。
