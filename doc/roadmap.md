# AINovel 后续工作路线图

更新时间：2026-07-23。本文是后续开发状态、顺序、门槛和待决策事项的唯一权威入口。代码与当前接口文档仍是实现事实来源。

## 状态定义

- `已完成`：代码、迁移、文档和约定验证已交付。
- `待运营验证`：功能已交付，但必须先积累真实数据。
- `门槛后立项`：指标达到前不得开始实现。
- `已确认后续`：方向已确认，仍受前置阶段阻塞。
- `独立排期`：不依赖主线门槛，可单独立项。
- `长期储备`：只有数据、模型或商业条件满足后才评估。
- `待决策候选`：仅保留证据，不代表承诺。
