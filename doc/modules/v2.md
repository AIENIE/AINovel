# v2 模块落地说明

## 定位

v2 在不破坏 `/api/v1/*` 的前提下增加上下文、风格、分析、版本、导出、模型和工作台体验能力。前端入口位于工作台与设置页，API 封装统一在 `frontend/src/lib/api-client.ts`。

## 模块清单

| 模块 | Controller | 主要前端入口 |
| --- | --- | --- |
| 上下文与图谱 | `V2ContextController` | 知识库、知识图谱 |
| 风格与角色声音 | `V2StyleController` | 设置-风格画像 |
| Beta Reader 与连续性 | `V2AnalysisController` | 工作台质量相关入口 |
| 文本、drift 与剧情质量 | quality Controllers | 稿件右侧剧情与质量分析页 |
| 版本与分支 | `V2VersionController` | 稿件版本侧栏 |
| 导出 | `V2ExportController` | 稿件导出侧栏 |
| 模型偏好 | `V2ModelController` | 设置-模型偏好 |
| 布局、会话、目标、快捷键 | `V2WorkspaceController` | 设置-工作台体验 |

## 关键行为

- 第一次访问版本功能时会初始化主分支和初始快照。
- 版本、分支、导出、Lorebook、图谱关系、风格画像、角色声音、写作会话和快捷键均持久化。
- 所有资源访问复用当前用户、故事和稿件的所有者校验；管理员模型路由接口额外要求管理员权限。
- 文本诊断、剧情诊断和长篇巡检不判断作者归因；候选修订必须由用户明确采纳。

接口详情见 `doc/api/v2-*.md`。
