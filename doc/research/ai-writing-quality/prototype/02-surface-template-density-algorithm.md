# 表层模板密度算法：Surface Template Density Algorithm

## 目标

检测小说文本中高频出现的 AI味表层模式：固定句式、动作链、器官轮班、意象套件、升华收尾、抽象情绪总结等。

注意：本算法不能单独证明 AI 使用，只能输出表层模板化风险。

## 输入

- 小说片段或章节文本。
- 可选：题材、平台、叙述人称、角色列表。
- 可选：pattern library，包括句式、动作、意象、收尾、商务/作文腔等词表。

## 输出

```json
{
  "surface_template_score": 0-100,
  "subscores": {
    "phrase_pattern_density": 0-100,
    "body_action_density": 0-100,
    "imagery_cliche_density": 0-100,
    "emotion_summary_density": 0-100,
    "ending_cliche_density": 0-100,
    "dialogue_tail_ratio": 0-100
  },
  "evidence": []
}
```

## 核心特征

### 1. 固定关联词句式

重点检测：

- 不是……而是……
- 不是……不是……而是……
- 没有……也没有……而是……
- 虽然……但是……
- 不是那种……而是一种更……
- 并不是……恰恰相反……

评分方法：

- 单次出现：低权重。
- 近距离重复：中权重。
- 多段共享同一逻辑骨架：高权重。
- 用于角色内心/叙述解释且替代具体行动：加权。

### 2. 语气描写模板

重点检测：

- 语气里带着……
- 声音里透着……
- 嗓音有些……
- 用一种……的口吻说
- 仿佛在阐述一个既定事实
- 平静得像在说今天天气怎么样

风险判断：

- 如果每句台词后都用抽象标签解释声音，提升风险。
- 如果语气标签没有对应台词内容或动作支撑，提升风险。

### 3. 身体动作链 / 器官轮班

重点检测：

- 指尖轻敲 / 指节泛白；
- 咬唇 / 喉咙发紧；
- 耳尖泛红 / 耳根红；
- 眼神一冷 / 眼中闪过光；
- 嘴角勾起弧度；
- 心咯噔一下；
- 无意识摩挲；
- 推眼镜；
- 拿起咖啡/烟但不喝/不吸。

评分方法：

- 统计每千字身体动作模板数。
- 统计同一动作跨角色复用次数。
- 统计动作是否产生后续影响。
- 与 Consistency 模块联动检测“动作与设定冲突”。

### 4. 意象套件 / 比喻八股

重点检测：

- 石子投入湖面 / 涟漪；
- 阳光透过窗户洒下光斑；
- 影子被拉得很长；
- 轻得像羽毛；
- 小兽般呜咽；
- 离水的鱼；
- 手术刀般精准；
- 浮木、投石、喉结等高频套件。

风险判断：

- 同一意象在多个无关场景重复使用。
- 比喻来源不符合角色经验。
- 比喻只提供“漂亮感”，不服务行动、情绪或场景信息。

### 5. 抽象情绪总结

重点检测：

- 某种更深的……
- 一种难以言喻的……
- 仿佛整个世界……
- 像背负了整个世界的重量；
- 撕心裂肺 / 喊碎世界 / 极致的无；
- 情绪词连续堆叠但缺少具体行为。

评分方法：

- 抽象情绪词密度。
- “仿佛/好像/似乎”密度。
- 情绪总结与具体行动比例。

### 6. 升华收尾套话

重点检测：

- 这就够了；
- 夜还很长；
- 不急，慢慢来；
- 才刚刚开始；
- 此刻无人知晓；
- 传奇永不落幕；
- 一切才刚刚开始。

风险判断：

- 章节结尾反复用同一类留白。
- 收尾没有具体情节钩子，只靠抽象气氛。

### 7. 对话尾巴描写比例

统计：

- 台词后是否总跟神态/心理/环境解释。
- 每句台词后的说明平均字数。
- 对话推进信息量与尾巴描写长度比例。

高风险表现：

> “我知道了。”她轻声说，声音里带着一丝不易察觉的颤抖，指尖无意识地摩挲着杯沿，仿佛那是她此刻唯一能够抓住的东西。

如果每句对话都这样，节奏会迟滞，形成 AI味。

## 密度与聚集评分

建议按窗口计算，例如每 500 字 / 1000 字：

```text
pattern_density = pattern_hits / token_count
cluster_score = max_hits_in_sliding_window
repetition_score = repeated_pattern_types / total_pattern_types
```

最终分数不要只看总命中数，而要看：

- 是否聚集在同一段；
- 是否反复使用同一结构；
- 是否跨角色复用；
- 是否与其他模块共振。

## 降权规则

以下情况应降权：

1. 单次出现。
2. 出现在符合文体的位置，如议论文使用“综上所述”。
3. 角色本身就有固定口癖，且文本有意识塑造。
4. 题材传统表达，如古早网文惯用动作。
5. 高密度但有明确讽刺/反讽目的。

## 升权规则

以下情况应升权：

1. 同一 pattern 在短窗口内重复。
2. 多种 pattern 同时出现。
3. 角色语域不贴。
4. 伴随事实矛盾或设定错配。
5. 出现在关键情绪/高潮段，替代了真正行动和选择。

## 修改建议生成

表层模板的修改不能只做同义词替换。

优先策略：

1. 删除无信息量情绪总结。
2. 把抽象语气改为台词节奏、停顿和选择。
3. 把通用动作换成角色专属动作。
4. 把通用比喻换成角色经验来源。
5. 保留必要的简洁表达，不要过度文学化。
