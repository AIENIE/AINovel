# 呼吸感 / 主体聚焦 / 节奏算法：Breath-Focus-Pacing Algorithm

## 目标

检测小说是否存在读者所说的“焦躁感”“没有呼吸感”“事件传送带”“几十章就看累”的问题。

该模块不是检测句子像不像 AI，而是检测叙事组织是否缺少主次、缓冲、后果消化和非数值成长。

## 输入

- 单章或多章文本。
- 章节大纲或事件列表，可由模型自动抽取。
- 角色状态表。
- 平台/题材基线：番茄快节奏、晋江关系推进、同人情绪沉淀等。

## 输出

```json
{
  "breath_focus_risk_score": 0-100,
  "subscores": {
    "event_conveyor_belt": 0-100,
    "aftermath_absence": 0-100,
    "focus_diffusion": 0-100,
    "ineffective_buffer": 0-100,
    "non_stat_growth_absence": 0-100,
    "world_reactivity_absence": 0-100
  },
  "diagnosis": [],
  "repair_tasks": []
}
```

## 核心概念

### 1. Breath / Focus 呼吸感与主体聚焦

好文本不是每个意象、每个动作、每个情绪都平均展开。它会有主次：

- 当前场景的核心对象是什么？
- 当前角色注意力在哪里？
- 哪个动作最重要？
- 哪些环境细节只需轻轻带过？

AI味常见问题：

- 一段里每个东西都被写得很满；
- 意象轮番上场；
- 读者注意力被不断打断；
- 看起来细，但没有焦点。

检测方法：

- 抽取每段核心动作/目标。
- 统计修饰性描写分散到多少对象。
- 检查段落是否存在明确中心。

### 2. Event Conveyor Belt 事件传送带

表现：

- 一个事件刚结束，下一个事件立刻来。
- 主角永远处在高强度搏命状态。
- 世界只围绕主角刷新冲突。
- 没有修整、恢复、生活纹理、关系沉淀。

检测方法：

1. 自动抽取事件列表。
2. 标注每个事件的起点、高潮、解决。
3. 检查事件之间是否存在 aftermath 段。
4. 统计连续高刺激事件数量。

### 3. Aftermath Coverage 后果覆盖

一个事件结束后，至少应回答部分问题：

- 谁受损？
- 谁得利？
- 资源/地位/关系怎么变？
- 主角学到了什么？
- 敌人如何反应？
- 下一目标如何从后果中长出来？

高风险：

- 事件解决后没有任何代价或余波。
- 下一事件像刷新任务，与上一事件没有因果关系。

### 4. Ineffective Daily Life Buffer 无效日常缓冲

缓冲不是“加七八章日常”。

无效缓冲：

- 纯聊天；
- 纯吃饭逛街；
- 只重复角色情绪；
- 不改变关系、不推进目标、不埋钩子。

有效缓冲：

- 处理上一事件后果；
- 沉淀关系；
- 显示主角内部变化；
- 重置短期目标；
- 让下一事件从旧后果中自然出现。

### 5. Non-Stat Character Growth 非数值成长

AI升级文常见问题：

- 主角只有等级/战力/财富成长；
- 判断方式不变；
- 恐惧不变；
- 关系处理不变；
- 价值观不变；
- 每个事件只是刷经验。

检测方法：

- 比较事件前后角色目标、恐惧、关系、策略是否变化。
- 如果连续多个事件只改变外部数值，风险升高。

### 6. World Reactivity 世界反应性

好长篇中，世界不是任务生成器。

检查：

- 配角是否有独立目标；
- 组织是否会根据主角行为调整；
- 旧事件是否改变社会关系；
- 敌人是否学习；
- 非主角人物是否在场外行动。

高风险：

- 所有人只在主角面前刷新事件；
- 配角只负责惊叹、误会、表达情绪；
- 世界没有记忆。

## 评分建议

```text
breath_focus_risk =
  0.25 * event_conveyor_belt +
  0.20 * aftermath_absence +
  0.15 * focus_diffusion +
  0.15 * non_stat_growth_absence +
  0.15 * world_reactivity_absence +
  0.10 * ineffective_buffer
```

平台修正：

- 番茄等快节奏平台允许更短缓冲，但不能无后果。
- 晋江/同人更看重关系沉淀，缓冲权重可提高。
- 短篇允许压缩 aftermath，但关键情绪必须落地。

## 修改策略

### 如果是事件传送带

补：

1. 上一事件造成的具体后果；
2. 角色关系变化；
3. 主角策略变化；
4. 下一事件从后果中生长的钩子。

### 如果是无效日常

不要增加闲聊，改成“带剧情锚定的低冲突段”。

### 如果是主体涣散

删掉同权展开的次要意象，保留当前角色真正会注意到的东西。

### 如果是固定 NPC 主角

为主角补一条非数值变化：

- 他开始害怕某种代价；
- 他改变处理关系的方式；
- 他意识到旧策略的局限；
- 他主动放弃一个爽点换取长期目标。
