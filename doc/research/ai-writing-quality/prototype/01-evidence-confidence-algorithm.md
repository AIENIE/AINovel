# 证据等级算法：Evidence Confidence Algorithm

## 目标

控制整个系统的安全边界：算法只评价文本呈现出的 slop / 模板化 / 工业化风险，不轻易推断作者是否使用 AI。

本模块先于其他模块运行，或在最终报告阶段统一收敛所有证据。

## 核心输出

```json
{
  "evidence_level": "E1|E2|E3|E4",
  "confidence_label": "low|medium|high|very_high",
  "allowed_claim": "允许输出的最强措辞",
  "forbidden_claims": ["禁止输出的措辞"],
  "evidence_items": []
}
```

## 证据等级

### E4：元提示 / 生成过程泄漏

最高置信，说明文本中出现了非正文生产痕迹。

典型证据：

- “以下是修改后的版本”
- “好的，这就为您……”
- “用户说过……”
- “根据你的大纲……”
- “指令：”
- “AI回复：”
- Markdown 结构残留：`###`、`---`、加粗列表等出现在小说正文中
- 角色名纠错残留：“苏晚——不，林子凡！”
- 生成过程左右互搏：“他是A？不对，不是，是B？也不是……”

允许措辞：

- “文本中存在明显非正文/生成过程残留。”
- “这属于高置信文本生产痕迹。”

仍不建议措辞：

- “作者一定用了某个具体模型。”
- “作者恶意使用 AI。”

### E3：结构性矛盾 / 设定错配

较高置信，说明文本世界内部出现硬错误。

典型证据：

- 角色没眼镜却推眼镜。
- 明确中国背景却持续出现樱花/百褶裙等外来模板。
- 刚说货架空，又说从货架拿到水。
- A 的经历/能力套到 B 的名字上。
- 前文设定的限制在后文被无视。
- 禁止新增设定后仍出现神秘仪器、异能、后遗症等。

允许措辞：

- “存在设定服从/角色绑定/局部连贯性风险。”
- “这类错误常见于自动扩写或低质量流水线文本，但也可能来自人工疏漏。”

### E2：多信号共振

中等置信，指多个弱信号在局部聚集。

典型组合：

- 固定句式高密度 + 动作链重复 + 抽象情绪总结。
- 比喻套件 + 升华收尾 + 角色语域不贴。
- 对话后尾巴描写过多 + 微动作膨胀 + 信息增量低。
- 长篇后段突然形容词密度上升 + 伏笔回收下降 + 事件机械续接。

允许措辞：

- “该片段呈现较明显模板化/AI味风险。”
- “多个信号共振，建议优先修改。”

禁止措辞：

- “可以确定是 AI 写的。”

### E1：单点体感 / 黑名单命中

最低置信，只能作为提示。

典型证据：

- 单次出现“不是……而是……”。
- 单次“指节泛白”。
- 单个“夜还很长”。
- 读者觉得“像 AI”“看着累”。

允许措辞：

- “这里有轻微模板化风险。”
- “单点证据不足，需要结合上下文。”

禁止措辞：

- 任何作者使用 AI 的推断。

## 汇总规则

### 最高证据优先

若出现 E4，整体 evidence_level 至少为 E4。
若无 E4 但出现多个 E3，整体可为 E3。
若主要是 E2 共振，整体为 E2。
若只有单点词句，整体为 E1。

### 共振升级

多个 E1 如果在短窗口内密集出现，可以升级为 E2。

例：500字内出现：

- 3次“不是……而是……”；
- 2次指节/喉咙/眼神动作链；
- 1个升华收尾；
- 且角色语域不贴。

则不是简单 E1，而是 E2 多信号共振。

### 替代解释要求

E1-E3 报告必须列出替代解释：

- 传统网文俗套；
- 人工低水平写作；
- 工作室公式化；
- 题材/平台惯例；
- 作者个人文风。

E4 可以不强调替代解释，但仍要避免恶意归因。

## 报告措辞模板

### 安全说法

> 该片段呈现出较高的模板化风险，主要证据是固定句式、动作链重复和角色语域不贴。当前证据只能说明文本层面的 slop 风险，不能单独证明作者使用了 AI。

### 禁止说法

> 这就是 AI 写的。
> 作者从这里开始用 AI。
> 这篇文是某模型生成的。

## 修改优先级关联

证据等级越高，修改优先级越高：

1. E4：必须删除或重写。
2. E3：必须修复设定/逻辑。
3. E2：按共振区域重写。
4. E1：可局部替换或观察是否形成密度问题。
