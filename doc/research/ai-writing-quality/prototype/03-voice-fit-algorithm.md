# 语域适配算法：Voice-Fit Algorithm

## 目标

判断文本中的词句、比喻、叙述腔、台词是否符合角色、叙述者、时代、题材、平台和场景。

这是防止“黑名单误判”的关键模块。很多词不是错，而是放错了人、放错了文本类型、放错了场景。

## 输入

- 文本片段。
- 角色档案：年龄、身份、教育程度、地域、职业、性格、关系、时代背景。
- 叙述视角：第一人称、第三人称有限、全知、同人原作视角等。
- 题材/平台：男频爽文、女频、同人、古言、现言、悬疑、番茄、晋江、LOFTER 等。

## 输出

```json
{
  "voice_fit_score": 0-100,
  "risk_items": [
    {
      "text_span": "...",
      "speaker_or_narrator": "...",
      "risk_type": "register_mismatch|character_mismatch|genre_mismatch|era_mismatch|fandom_mismatch",
      "reason": "...",
      "suggestion": "..."
    }
  ]
}
```

分数越高表示语域不适配风险越高。

## 主要检测类型

### 1. 文体错位

例子：

- 小说正文突然出现“综上所述”。
- 角色内心独白像商业复盘：“底层逻辑”“闭环”“赋能”。
- 古代角色说出现代管理学词汇。
- 高中生说话像 AI 长文分析：“这本质上涉及的是……”

判断原则：

- 在议论文/说明文里正常的词，放到小说人物意识里可能突兀。
- 在旁白中可接受的分析词，放到台词中可能不自然。

### 2. 角色身份错位

检测台词和叙述是否符合角色：

- 年龄；
- 职业；
- 教育背景；
- 社会阶层；
- 情绪状态；
- 与对话对象的关系。

例子：

- 粗粝江湖人突然使用精致文艺腔。
- 冷静理工男频繁说“某种难以言喻的情绪”。
- 小孩说话像成年人总结人生。
- 所有角色共享同一种“AI礼貌腔”。

### 3. 时代/世界观错位

例子：

- 古代世界出现现代心理学/商业词。
- 中国校园无故出现樱花、百褶裙、日式社团模板。
- 西幻角色使用中式官场话。
- 修仙世界中突然出现“系统性优化”“数据闭环”但不是设定内术语。

### 4. 同人角色适配

同人文需要单独加强：

- 是否贴合原作角色说话方式；
- 是否尊重原作关系史；
- 是否加入不属于原作世界的高科技/宏大理论；
- 是否只复现 CP 套话，而没有角色内核。

高风险表现：

- 表面叫对了名字，关系标签也对，但角色做的选择不像原作中的他。
- 热门 CP 的平均梗堆叠，缺少具体作者理解。

### 5. 叙述视角错位

例子：

- 第三人称有限视角突然知道所有人的心理。
- 第一人称角色使用自己不可能知道的信息。
- 紧张战斗中突然长篇主题分析。
- 角色明明处在危险里，旁白却开始抽象升华。

## 评分方法

建议从 5 个子分数合成：

```text
voice_fit_risk =
  0.25 * register_mismatch +
  0.25 * character_mismatch +
  0.20 * genre_platform_mismatch +
  0.15 * era_worldview_mismatch +
  0.15 * pov_mismatch
```

同人文可提高 `character_mismatch` 和 `fandom_mismatch` 权重。

## 与表层模板算法的关系

表层模板命中后，必须问：

1. 这个词/句在此文体中正常吗？
2. 这个角色会这么说吗？
3. 这个叙述者会这么观察吗？
4. 这个题材/平台允许这种表达吗？
5. 它是否高密度重复？

如果答案是“适配”，则降低风险。
如果“不适配”，则升高风险。

## 修改策略

### 不要只换词

错误修法：

- 把“底层逻辑”换成“根本原因”。

如果角色本来不会做抽象分析，换词仍然假。

### 正确修法

1. 先确定角色此刻真正想达成什么。
2. 根据角色身份决定他说多直、多绕、多粗、多含蓄。
3. 用角色熟悉的经验系统替换比喻。
4. 用行为和选择代替旁白解释。

## 示例

原句：

> 她意识到，这并不是单纯的背叛，而是一种更深层的权力结构重组。

风险：

- 如果是政治谋士旁白，可接受。
- 如果是普通少女失恋内心，语域错位。

改法：

> 她忽然明白，自己不是被他丢下了。更糟的是，从一开始，他就没把她放进那个“我们”里。

这里保留了认知变化，但换成角色能承受的语言。
