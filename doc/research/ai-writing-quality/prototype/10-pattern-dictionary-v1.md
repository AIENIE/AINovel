# Pattern Dictionary v1：中文小说 AI味 / Slop 特征字典

本文件把前期研究中收集到的 AI味、模板化、工业化文本特征整理成可维护字典。它不是“命中即判定 AI”的黑名单，而是供评分算法使用的弱/中/强信号库。

## 使用原则

1. **单点命中只算弱信号**：一个词、一句话不能证明 AI 使用。
2. **密度、聚集、跨信号共振更重要**：短窗口内多类 pattern 同时出现，风险才升高。
3. **必须结合语域适配**：同一句话在议论文里正常，在小说台词中可能突兀。
4. **必须保留替代解释**：传统网文俗套、低水平人工、工作室公式都可能命中特征。
5. **可维护**：每个 pattern 应有类别、权重、误判风险、适用场景和修改建议。

## 字段建议

```yaml
id: P001
category: phrase_pattern | body_action | imagery | ending | meta_leak | contradiction | platform_formula
pattern: "不是……而是……"
match_type: regex | phrase | semantic | llm_judge
base_weight: 1-5
false_positive_risk: low | medium | high
notes: "为什么有风险"
repair_hint: "如何修改"
```

# A. 固定句式 / 关联词模板

## A001：不是……而是……

- 类别：phrase_pattern
- 基础权重：2
- 误判风险：高
- 风险说明：单次正常，高密度重复时形成 AI 解释腔；常用于把具体情绪抽象升级。
- 升权条件：500字内出现 3 次以上；与动作链/情绪总结共现；角色台词不该这么抽象。
- 降权条件：议论文、分析文、角色有明确辩论语气。
- 修改建议：改为角色具体行动、判断、停顿或选择。

## A002：不是……不是……而是……

- 类别：phrase_pattern
- 基础权重：3
- 误判风险：中
- 风险说明：多重否定更容易显得模型式“层层递进”。
- 修改建议：删掉否定铺垫，直接写角色真正感受到/决定的东西。

## A003：没有……也没有……而是……

- 类别：phrase_pattern
- 基础权重：2
- 误判风险：中高
- 风险说明：常用来制造“高级感”的排除式叙述。

## A004：仿佛在阐述一个既定事实

- 类别：phrase_pattern / tone_description
- 基础权重：3
- 误判风险：中
- 风险说明：AI常用来描述角色平静或压迫感，但容易变成通用气氛贴纸。

## A005：平静得像在说今天天气怎么样

- 类别：tone_description
- 基础权重：3
- 误判风险：中
- 风险说明：被多个用户点名的语气模板。

# B. 语气描写模板

## B001：语气里带着……
## B002：声音里透着……
## B003：嗓音有些……
## B004：语调变得……
## B005：用一种……的口吻说

- 类别：tone_description
- 基础权重：2
- 误判风险：高
- 升权条件：几乎每句台词后都跟一段语气/神态/心理解释；语气标签没有从台词内容中自然显现。
- 修改建议：用台词本身的词序、停顿、避而不答、转移话题来表现潜台词。

# C. 身体动作链 / 器官轮班

## C001：指尖轻敲 / 叩桌面

- 类别：body_action
- 基础权重：2
- 误判风险：中高
- 升权条件：多个角色都敲；动作不影响对话或节奏。

## C002：指节泛白 / 指尖泛白

- 类别：body_action
- 基础权重：3
- 误判风险：中
- 风险说明：用户高频点名；常被用于紧张、愤怒、隐忍等所有情绪。

## C003：死死咬住下唇

- 类别：body_action
- 基础权重：2
- 误判风险：中

## C004：喉咙发紧

- 类别：body_action
- 基础权重：2
- 误判风险：中

## C005：耳尖/耳根泛红

- 类别：body_action
- 基础权重：2
- 误判风险：中
- 适用：暧昧/羞耻场景高频，需看密度。

## C006：推眼镜

- 类别：body_action / consistency_check
- 基础权重：2；若角色无眼镜则升为 E3
- 误判风险：中
- 修改建议：检查角色是否真的有眼镜；否则改成符合角色习惯的动作。

## C007：拿起咖啡/烟但不喝/不吸

- 类别：body_action
- 基础权重：2
- 风险说明：常作为“沉默/压抑”模板动作。

## C008：嘴角勾起一个弧度

- 类别：body_action / expression
- 基础权重：3
- 风险说明：冷笑/玩味/压迫感通用模板。

## C009：眼中闪过某种光 / 瞳孔放大或收缩

- 类别：body_action / expression
- 基础权重：2
- 风险说明：器官轮班常见组件。

# D. 意象 / 比喻套件

## D001：石子投入湖面 / 涟漪

- 类别：imagery
- 基础权重：3
- 误判风险：中
- 风险说明：多平台反复出现的 AI八股意象。
- 修改建议：换成角色经验系统内的具体反应，不要只换另一个漂亮比喻。

## D002：影子被夕阳/月光/路灯拉得很长

- 类别：imagery / scene_ending
- 基础权重：3

## D003：阳光透过窗户洒进来，在地上投下光斑

- 类别：imagery / opening_scene
- 基础权重：2

## D004：轻得像羽毛

- 类别：imagery
- 基础权重：2

## D005：小兽般呜咽

- 类别：imagery / intimacy
- 基础权重：3

## D006：离水的鱼 / 煮熟的虾米

- 类别：imagery
- 基础权重：2

## D007：手术刀般精准 / 浮木 / 投石 / 喉结等八股比喻

- 类别：imagery_cluster

# E. 升华 / 收尾套话

## E001：这就够了
## E002：夜还很长
## E003：不急，慢慢来
## E004：才刚刚开始 / 游戏才刚刚开始
## E005：此刻，无人知晓
## E006：注定无法平静 / 无人能够预料
## E007：传奇永不落幕

- 类别：ending_cliche
- 基础权重：2-3
- 误判风险：中
- 升权条件：章节结尾反复出现；没有具体剧情钩子，只靠抽象留白。
- 修改建议：用具体未解决问题、角色新目标、后果回声替代抽象升华。

# F. 元提示 / 生成过程泄漏

## F001：以下是修改后的版本 / 好的，这就为您

- 基础权重：5

## F002：用户说过 / 根据你的大纲 / 指令

- 类别：meta_leak
- 基础权重：5
- 证据等级：E4

## F003：角色名纠错残留

例：“苏晚——不，林子凡！”

- 类别：generation_residue
- 基础权重：5
- 证据等级：E4

## F004：Markdown 残留

例：`###`、`---`、列表、加粗标记出现在小说正文。

- 类别：answer_residue
- 基础权重：4-5
- 证据等级：E4 或 E3，视上下文而定。

# G. 事实矛盾 / 设定冲突

## G001：近句事实互斥

例：拿了水 / 货架空 / 没拿到水。

- 类别：local_contradiction
- 基础权重：4
- 证据等级：E3

## G002：动作与角色设定冲突

例：没眼镜却推眼镜。

- 类别：setting_action_conflict
- 基础权重：4
- 证据等级：E3

## G003：世界观不当元素漂移

例：中国背景无故出现樱花、百褶裙；古代角色说现代商务词。

- 类别：worldview_mismatch
- 基础权重：3-4

## G004：未授权新增

例：突然神秘仪器、突然异能、外挂被擅自加后遗症。

- 类别：unauthorized_addition
- 基础权重：4

# H. 叙事机制 pattern

## H001：事件传送带

- 类别：narrative_mechanics
- 基础权重：4
- 表现：一个事件结束，下一个马上来，无修整、后果、关系沉淀。

## H002：无效日常缓冲

- 类别：pacing
- 基础权重：3
- 表现：连续聊天/吃饭/逛街，但不改变关系、不推进目标。

## H003：固定 NPC 主角

- 类别：character_growth
- 基础权重：4
- 表现：只有战力/等级成长，没有欲望、判断、关系、代价意识变化。

## H004：Prompt Assimilation Failure

- 类别：setting_assimilation
- 基础权重：4
- 表现：输入大量设定，正文只抓人名和关键词，仍按默认模板写。

## H005：Default Protagonist Drift

- 类别：character_drift
- 基础权重：4
- 表现：温和/闷骚/普通人设漂移成冷笑、霸道、冷酷爽文主角。

# I. 平台 / 商业公式 pattern

## I001：概念堆叠钩子

- 类别：platform_formula
- 基础权重：3
- 表现：短句内塞死亡、婚礼、祖传、神器、复仇、身份反转等多个爆点，但因果支撑不足。

## I002：前三章硬塞爽点/暧昧

- 类别：studio_formula
- 基础权重：3
- 注意：可能是人工工作室公式，不等同 AI。

## I003：半文收费 / 彩蛋截断 / 热圈批量覆盖

- 类别：monetization_slop
- 基础权重：3
- 适用：LOFTER/同人/短篇变现。

# J. 读者体感 pattern

## J001：看着累

- 类别：reader_fatigue
- 基础权重：1-3
- 注意：必须归因，不能直接判 AI。可能来源包括文字功底差、设定复杂、动作模板、事件传送带、情绪不激活。

## J002：油味

- 类别：emotional_activation / power_fantasy
- 基础权重：2-3
- 表现：廉价霸气、自我陶醉、姿态化装逼、人物关系服务主角光环。

## J003：没人味 / 没有灵魂

- 类别：human_trace
- 基础权重：1-3
- 注意：体感价值高，但证据等级低，需转换成具体证据：作者取舍、具体经验、角色选择是否缺失。
