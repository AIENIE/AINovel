# 作者痕迹算法：Human Trace Algorithm

## 目标

评估文本是否呈现出作者的具体经验、审美取舍、创作意图和不可替代选择。

该算法不是判断作者是不是人，而是判断文本中是否存在“人的取舍痕迹”。AI slop 常见问题是每句都合理、漂亮、流畅，但整体像平台平均值或模型平均答案。

## 输出

```json
{
  "human_trace_risk_score": 0-100,
  "positive_traces": [],
  "missing_traces": [],
  "repair_suggestions": []
}
```

分数越高表示作者痕迹缺失风险越高。

## 核心维度

### 1. 具体经验痕迹

正向信号：

- 具体但不炫耀的行业/地域/生活细节；
- 细节会影响角色行动，而不是装饰；
- 细节有取舍，不是百科式堆砌。

负向信号：

- 所有细节都泛泛而谈；
- 像知识库拼贴；
- 细节不影响剧情；
- 每个场景都使用通用环境模板。

### 2. 作者偏执 / 核心趣味

正向信号：

- 能看出作者特别想写的关系、爽点、痛点、场景或主题；
- 文本有稳定趣味，不只是追平台热点；
- 有些选择不完全“最优”，但有个人味。

负向信号：

- 所有设计都像平台爆款总结；
- 每个桥段都正确但无聊；
- 看不出作者为什么非写这个不可。

### 3. 角色不可替代选择

正向信号：

- 场景中的选择只有这个角色会做；
- 选择来自角色历史、欲望、恐惧和关系；
- 角色为选择付出代价。

负向信号：

- 换一个主角剧情也一样；
- 角色只按爽文默认逻辑行动；
- 冲突解决靠模板能力或外部刷新。

### 4. 理解而非排列

正向信号：

- 文本持续承认自己设定的后果；
- 情绪不是标签，而是由处境和选择长出来；
- 角色关系有记忆。

负向信号：

- 语句排列得像小说，但不理解自己在讲什么；
- 前一章发生的事不改变后一章；
- 主题靠旁白总结，不靠剧情呈现。

### 5. 适度不完美

正向信号：

- 文本保留适当个人节奏；
- 有独特但稳定的用词或观察；
- 不把所有句子抹成“正确作文”。

负向信号：

- 全篇过于平滑；
- 所有段落都像标准答案；
- 修改后没有错，也没有记忆点。

## 评分方法

```text
human_trace_missing_risk =
  0.25 * lack_of_specific_experience +
  0.25 * lack_of_authorial_intent +
  0.20 * replaceable_character_choices +
  0.20 * weak_consequence_understanding +
  0.10 * over_smoothed_style
```

## 与平台公式化的关系

平台公式化会压低 Human Trace：

- 为了过签硬塞暧昧；
- 为了首章爆点堆概念；
- 为了热圈变现复用 CP 套话；
- 为了短视频转化制造高光切片。

但这不一定是 AI，也可能是人工工业化。

## 修改策略

### 1. 找回作者真正想写的三件事

让作者回答：

1. 我最想看的关系是什么？
2. 我最想写的爽点/痛点是什么？
3. 这个故事里我最不想妥协的东西是什么？

把答案嵌回主线。

### 2. 为每场戏设置不可替代选择

问：

- 如果换一个主角，这场戏会不会一样？
- 这个角色为什么非这样做不可？
- 他的选择会让他失去什么？

### 3. 用具体经验替代通用漂亮话

不要增加百科资料。
只加入会改变角色判断、行动、关系的经验。

### 4. 保留个人怪癖

不要把所有句子都润成模型式流畅。
有些笨拙但有辨识度的表达可以保留。

## 报告措辞

允许：

- “这段文本作者取舍痕迹较弱，更多依赖通用桥段。”
- “角色选择可替代性较高，换成其他主角也基本成立。”

禁止：

- “没有人类作者的灵魂。”
- “这肯定不是人写的。”
