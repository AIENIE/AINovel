# 长篇中后段断层算法：Mid-book Drift Algorithm

## 目标

评估长篇小说是否存在“前面像人写/人工打磨，十万字或首秀后变得模板化、机械化、AI味增强”的风险。

该算法检测的是文风与叙事机制断层，不直接推断作者从某章开始使用 AI。

## 输入

至少需要多个章节窗口：

- 开头窗口：第1-3章；
- 早期窗口：3-5万字附近；
- 首秀/关键节点：10万字附近；
- 中期窗口：20万字附近；
- 最新窗口。

每个窗口建议 3000-10000 字。

## 输出

```json
{
  "midbook_drift_risk_score": 0-100,
  "drift_points": [
    {
      "from_window": "early",
      "to_window": "100k",
      "changed_metrics": [],
      "interpretation": "...",
      "safe_claim": "文风/叙事机制断层风险"
    }
  ],
  "metric_curves": {}
}
```

## 比较指标

### 1. 表层模板曲线

跨窗口比较：

- 固定句式密度；
- 动作链密度；
- 意象套件密度；
- 抽象情绪总结密度；
- 升华收尾密度；
- 形容词/赞美词密度。

风险：

- 某节点后突然上升。
- 早期较具体，后期变成通用华丽表达。

### 2. 因果与目标曲线

比较每个窗口：

- 本章目标是否明确；
- 行动是否由上一事件后果触发；
- 冲突解决是否产生新代价；
- 是否存在长期目标推进；
- 是否有读者期待管理。

风险：

- 后期目标变模糊；
- 事件靠外部刷新；
- 情节像机械续接。

### 3. 角色稳定曲线

比较：

- 主角语气是否稳定；
- 角色行为逻辑是否一致；
- 配角是否还保留独立目标；
- 关系是否有阶段性变化；
- 主角是否有非数值成长。

风险：

- 后期角色只剩标签；
- 配角只负责惊叹/误会/表达情绪；
- 主角从具体人格漂移成默认爽文主角。

### 4. 伏笔与记忆曲线

比较：

- 前文伏笔是否回收；
- 旧设定是否继续影响行动；
- 道具/能力/限制是否被记住；
- 角色关系史是否被承认。

风险：

- 后文像新开一本；
- 回收伏笔时像临时新增；
- 中段重复解释或遗忘。

### 5. 节奏呼吸曲线

比较：

- 事件间隔；
- aftermath 段比例；
- 日常/缓冲段是否有剧情锚定；
- 连续高刺激事件数量。

风险：

- 后期持续事件传送带；
- 或者后期大量无效日常填字数。

## 断层判断

断层不是“后期质量下降”这么简单。要看是否同时出现：

1. 表层模板密度上升；
2. 叙事因果质量下降；
3. 角色稳定性下降；
4. 设定/伏笔记忆下降；
5. 节奏从因果推进变成机械续接。

满足越多，风险越高。

## 安全措辞

允许：

- “10万字附近出现文风和叙事机制断层。”
- “后期模板化风险显著升高。”
- “该变化可能来自 AI辅助比例变化、赶稿、换写手、作者疲劳或平台节奏压力。”

禁止：

- “作者从第X章开始用 AI。”
- “第X章以后一定是机器写的。”

## 修改策略

### 如果后期表层模板上升

- 删除高频套话；
- 恢复早期角色语气；
- 从早期文本提取作者风格锚点。

### 如果后期因果弱化

- 为每个大事件补“上一事件后果 → 当前目标 → 行动代价 → 新后果”。

### 如果角色漂移

- 建立角色绑定表；
- 列出该角色不会做/不会说的内容；
- 重写关键台词和选择。

### 如果伏笔遗忘

- 做伏笔生命周期表：planned → planted → echoed → harvested。
- 后文每次推进前检查旧伏笔是否应回声。

## 可视化建议

生成曲线：

- Template Density over Chapters
- Causal Coherence over Chapters
- Role Stability over Chapters
- Breath Score over Chapters
- Retention Risk over Chapters

这样更适合长篇诊断，而不是只给单点结论。
