# 一致性与设定吸收算法：Consistency & Assimilation Algorithm

## 目标

检测文本是否真正服从既有设定、角色档案和场景事实，而不是只抓人名和关键词后按默认模板续写。

该模块对应 AI slop 中较深层的问题：局部事实矛盾、角色绑定失败、未授权新增、设定吸收失败。

## 输入

- 文本片段或章节。
- 上文摘要。
- 角色设定表。
- 世界观规则。
- 当前场景状态：地点、时间、道具、人物、目标、限制。
- 禁止新增项或必须保留项。

## 输出

```json
{
  "consistency_risk_score": 0-100,
  "assimilation_risk_score": 0-100,
  "issues": [
    {
      "type": "local_contradiction|character_binding_error|unauthorized_addition|constraint_violation|setting_ignored",
      "evidence": "...",
      "expected_state": "...",
      "actual_text": "...",
      "severity": "low|medium|high"
    }
  ]
}
```

## 检测类型

### 1. 近句事实矛盾

例子：

- “她从货架上拿了一瓶水，货架上是空的，她没拿到水。”
- “门已经被锁死”，下一句角色直接推门进来。
- 刚说天黑，下一句阳光洒进来。

方法：

- 抽取局部状态：人物、动作、道具、空间、时间。
- 建立短窗口事实表。
- 检测互斥状态。

### 2. 动作与角色设定冲突

例子：

- 没眼镜角色推眼镜。
- 断臂角色双手抱胸。
- 不能说话的角色开口长篇解释。
- 不会武的角色突然打出专业招式。

方法：

- 从角色档案抽取身体、能力、习惯、限制。
- 对动作进行可行性检查。

### 3. 角色绑定失败

例子：

- A 的童年经历被写到 B 身上。
- 角色名字正确，但性格变成另一个模板。
- 毒舌萝莉被写回普通撒娇萝莉。
- 暖心总裁被写成标准冷酷霸总。

方法：

- 为每个角色建立稳定特征向量：目标、恐惧、口癖、能力、关系、底线。
- 检查当前文本是否保持绑定。
- 特别保护“反刻板设定”。

### 4. 未授权新增

例子：

- 突然出现神秘仪器。
- 给外挂擅自添加后遗症。
- 给角色添加原设没有的异能。
- 在现实题材中突然引入玄幻解释。

方法：

- 区分“合理扩展”和“破坏性新增”。
- 如果新增改变剧情规则、角色能力、冲突解决方式，则高风险。

### 5. Prompt Assimilation Failure 设定吸收失败

典型表现：

- 输入大量设定，正文只使用人名和几个关键词。
- 世界规则没有影响角色决策。
- 角色特质只在介绍中出现，行动中消失。
- 不同题材最后都被写成同一种“爽文默认主角”。

评估问题：

1. 设定限制是否真的造成行动代价？
2. 角色特质是否改变了他的选择？
3. 世界规则是否影响冲突解决？
4. 前文关系是否影响对话和站位？
5. 反套路设定是否被保留？

## 权重建议

| 问题 | 权重 |
|---|---:|
| 元提示泄漏 | 极高，交给 Evidence 模块 E4 |
| 近句事实矛盾 | 高 |
| 动作设定冲突 | 高 |
| 角色绑定失败 | 高 |
| 未授权新增设定 | 高 |
| 设定只浅层出现 | 中高 |
| 小道具轻微错漏 | 中低 |

## 与改写 workflow 的关系

一致性问题必须优先于文风问题修复。

修改顺序：

1. 修硬矛盾。
2. 删除未授权新增。
3. 恢复角色绑定。
4. 让设定影响行动和代价。
5. 最后再润色语言。

## 示例

设定：主角是温和、怕冲突、靠观察解决问题的人。

AI味文本：

> 他冷笑一声，眼神如刀，抬手便将对方震飞出去。

问题：

- Default Protagonist Drift：被拉回冷酷爽文主角。
- 角色行为逻辑不符。
- 如果主角没有武力，还存在能力冲突。

修复方向：

> 他没有立刻反驳，只是把对方刚才漏说的那半句话重复了一遍。屋里安静下来时，他才意识到，自己手心全是汗。

这里保留冲突，但让解决方式符合角色。
