# 诊断与改写 Workflow Prompt Templates v1

本文件把算法设计转为可直接给 AI 使用的提示词模板。目标是让 AI 基于 pattern library 做证据驱动诊断和针对性修改，而不是泛泛“润色去AI味”。

---

# Template 1：安全诊断总提示词

```text
你是中文小说文本诊断助手。你的任务不是判断作者是否使用 AI，而是评估文本呈现出的模板化、工业化、AI味/slop 风险，并给出可修改的证据。

必须遵守：
1. 不说“作者一定用了AI”。
2. 每个判断必须引用原文证据。
3. 单个黑名单词只能作为弱信号。
4. 必须列出替代解释，如传统网文俗套、人工低水平写作、平台公式化、作者个人风格。
5. 输出修改建议时，先修设定/因果/角色，再修词句。

请按以下结构输出：
- 总览：风险等级、证据等级、安全结论
- 分层评分：表层模板、语域适配、一致性、呼吸感、作者痕迹
- 证据表：原文/类型/等级/说明/替代解释/建议
- 修改优先级
- 改写任务
```

# Template 2：表层模板诊断

```text
请只诊断以下文本的“表层模板化风险”，不要判断作者是否使用 AI。

重点检查：
1. 固定句式：不是……而是……、没有……而是……等。
2. 语气描写：语气里带着、声音里透着、像在说今天天气等。
3. 身体动作链：指节泛白、喉咙发紧、咬唇、耳尖泛红、推眼镜等。
4. 意象套件：石子涟漪、影子拉长、阳光光斑、羽毛、小兽等。
5. 升华收尾：夜还很长、才刚刚开始、无人知晓等。
6. 对话尾巴：每句台词后是否都拖神态/心理/环境解释。

输出表格：
| 原文 | Pattern类型 | 密度/共现说明 | 证据等级 | 修改建议 |

文本：
{{TEXT}}
```

# Template 3：语域适配诊断

```text
请诊断文本中是否存在角色/叙述者语域不适配。不要只看词是否高级，而要判断这个角色、这个视角、这个时代、这个题材是否会这样表达。

已知信息：
- 题材/平台：{{GENRE_PLATFORM}}
- 叙述视角：{{POV}}
- 角色档案：{{CHARACTER_PROFILES}}
- 世界观/时代：{{WORLD_SETTING}}

检查：
1. 台词是否像这个角色会说的话。
2. 内心独白是否符合角色教育、身份、情绪状态。
3. 旁白是否越界成 AI 分析腔/议论文腔。
4. 是否有时代/世界观错位词。
5. 同人文是否贴原作角色。

输出：
- 不适配片段
- 为什么不适配
- 可能替代解释
- 如何改成更贴角色的表达

文本：
{{TEXT}}
```

# Template 6：长篇中后段断层诊断

```text
请比较以下多个章节窗口，诊断是否存在中后段文风/叙事机制断层风险。

注意：你不能说“作者从这里开始用AI”。只能说“文本在某阶段出现模板化/叙事机制断层风险”。

窗口：
{{WINDOWS}}

请比较：
1. 固定句式、动作链、抽象情绪、形容词密度是否变化。
2. 角色语气和行为逻辑是否漂移。
3. 事件推进是否从因果链变成机械续接。
4. 伏笔、道具、关系是否被遗忘。
5. 主角是否从具体人格变成默认爽文主角。
6. 哪个窗口开始出现明显变化。

输出：
- 指标曲线文字总结
- 断层证据表
- 替代解释：赶稿、换写手、剧情高潮、平台节奏、AI辅助比例变化等
- 修复任务
```

# Template 7：Human Trace / 作者痕迹诊断

```text
请评估文本是否缺少作者痕迹。注意：这不是判断作者是不是人，而是看文本是否有具体经验、稳定趣味、不可替代选择和真实取舍。

检查：
1. 是否有具体但不炫耀的经验细节。
2. 是否能看出作者真正想写的关系/爽点/痛点。
3. 角色选择是否不可替代，还是换个主角也一样。
4. 事件是否持续承认后果。
5. 文风是否过度正确平滑，缺少个人记忆点。

输出：
- 正向作者痕迹
- 缺失的作者痕迹
- 哪些段落最像平台平均模板
- 如何补回作者取舍

文本：
{{TEXT}}
```

# Template 8：证据驱动改写

```text
你将根据诊断报告改写文本。不要泛泛润色，不要只替换黑名单词。

优先级：
1. 修复元泄漏/事实矛盾/设定冲突。
2. 修复角色行为逻辑。
3. 补事件后果、代价、关系变化。
4. 修复角色语域和潜台词。
5. 降低动作链、意象套件、升华套话密度。
6. 保留作者原有有效信息和风格，不要抹平成标准作文。

输入：
- 原文：{{TEXT}}
- 诊断证据：{{EVIDENCE_TABLE}}
- 角色档案：{{CHARACTER_PROFILES}}
- 必须保留：{{MUST_KEEP}}
- 禁止新增：{{DO_NOT_ADD}}

输出：
1. 改写策略，逐条对应证据。
2. 改写版本。
3. 修改说明：每处修改解决了什么问题。
4. 剩余风险：哪些需要人工确认。
```

# Template 9：缓冲段生成

```text
请为以下事件之后设计一个“有剧情锚定的缓冲段”。不要写纯日常水文。

上一事件：{{PREVIOUS_EVENT}}
事件后果：{{CONSEQUENCES}}
角色当前状态：{{CHARACTER_STATE}}
下一主线目标：{{NEXT_GOAL}}
关系线：{{RELATIONSHIP_THREADS}}
字数范围：{{LENGTH}}

缓冲段必须完成至少三项：
1. 处理上一事件代价。
2. 改变一个角色关系或权力位置。
3. 展示主角非数值成长。
4. 重置短期目标。
5. 埋下下一事件钩子。
6. 让读者情绪落地。

禁止：
- 纯聊天无推进。
- 新增无关敌人。
- 用“夜还很长/才刚刚开始”等抽象收尾。
```
