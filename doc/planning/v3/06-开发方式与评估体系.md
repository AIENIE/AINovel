## 1. 总体推进顺序与理由

**推荐顺序：G2 → G1 →（G3 + G4 摄入共建）→ G4 完全体**

```
阶段一  G2 Step 1 盲测闭环与门槛验证                    ← 已交付，等待真实样本
阶段二  G2 方案A（节拍规划+多候选竞技场）               ← 达到门槛后立项
阶段三  G1 方案A（P0 已交付；正文步挂 G2 管线）         ← P1 在 G2 A 后推进
阶段四  G3 方案A+B（摄入管线+拆解+技法卡） ┐
        G4 一期（摄入+指纹+状态库+RAG 续写）┘ 摄入管线共建，一次投资两处收益
阶段五  G4 二期（规划器+契约+保真闭环）= 旗舰功能完全体
持续    G2 方案B（模型解锁后增量迭代）；方案C 数据积累达标后另行立项
```

理由：

1. **G2 是分母**：向导（G1）产出的初稿、技法卡（G3）作用的正文、续写（G4）的起草全部流经生成管线。管线不升级，其余目标做得越顺，产出平庸文本的速度越快。
2. **G1 回报最快**：G2 之后做 G1，向导每一步天然享受多候选与节拍规划，"轻松"与"好"同时兑现。
3. **G3/G4 共享摄入管线**（分章解析、向量化、逐章标注），必须同期设计、一次建成。
4. **G4 最后收口**：它是对 G2（起草质量）、G3（摄入）和现有状态记忆能力的总集成，前置件齐备后风险最低。

**前置偿还项**：
- 在 G1 `async_jobs`、租约和恢复机制上提取通用异步任务边界（G3/G4 摄入的共同前提）；
- AI 网关 gRPC 透传采样参数（G2 方案 A 的唯一跨服务改动）；
- KAG 图谱构建完整化可延后（G4 账本有 Lorebook + JSON 降级方案，见 04§4.3）。

## 2. 每目标的统一分期模板

| 期 | 内容 | 出口标准 |
|----|------|----------|
| P0 | 数据模型 + 核心 Service + 最小可用链路（内部可跑通） | 内部演示 + 离线评估达标 |
| P1 | 管线集成（PromptAssembly/QualityGate/StyleContext 触点）+ 计费接入 | 灰度用户可用；遥测落库完整 |
| P2 | 前端产品化 + 用户引导 | 正式发布；A/B 实验开启 |
| P3 | 基于遥测与实验结果调优（阈值、提示词版本、成本参数） | 达成该目标的核心指标 |

每期结束设**决策门**：核心指标未达标时，先调参迭代（提示词版本化使回滚廉价），连续两轮不达标则回到方案对比表评估切换方案（如 G2 A→B）。

## 3. 评估体系（核心章节）

### 3.1 基础设施

1. **提示词版本化**：`PromptTemplateRegistry`（00§4）——任何提示词改动都是一个可回滚、可对比的版本；实验分桶按 userId 哈希。
2. **统一质量信号**：扩展现有 `SlopQualitySignals`，所有生成路径（fast/crafted/续写）落同构遥测，保证跨方案可比。
3. **盲测面板**：首版只比较 fast/crafted 匿名文本；人类写作对照锚点属于后续独立实验，不能混入 G2 Step 1 门槛。
4. **自动指标服务**：本地统计（distinct-n、句长分布、对话占比、虚词分布）零成本全量跑；LLM 评审类指标抽样跑且**评审模型与生成模型异源**（LLM 评审偏好自家文本，见 05§3）。

### 3.2 各目标指标总表

| 目标 | 指标 | 类型 | 基线来源 | 目标值 |
|------|------|------|----------|--------|
| G2 | slop 风险分中位数 | 自动 | `slop_quality_runs` 近 90 天 | -20% |
| G2 | 触发保守修订比例 | 自动 | 同上 | -30% |
| G2 Step 1 | 盲测精雕胜率（crafted vs fast） | 人工 | `g2_evaluation_*` | ≥55%，且 100 票 / 20 样本对 / 10 评审 |
| G2 | 同大纲 10 次生成的 distinct-3 / 语义距离 | 自动 | 上线前离线测定 | 显著高于 fast（p<0.05） |
| G2 | 节拍三要素（欲望/冲突/代价）完备率 | LLM 审计 | 新建 | ≥90% |
| G1 | 新建故事→首章完成中位时长 | 自动 | 现有行为日志 | -50% |
| G1 | 新建后 24h 产出正文的用户占比 | 自动 | 同上 | +20pp |
| G1 | 单章生成后人工修改比例 | 自动（编辑距离）| 需按当前版本建立 | <35%（候选目标，与 G2 共担） |
| G3 | 导入→查看报告→启用技法卡转化漏斗 | 自动 | 新建 | 报告→启用 ≥30% |
| G3 | 拆解结论与人工标注一致率（抽样） | 人工 | 新建 | ≥80% |
| G3 | 技法卡启用后对应维度盲测胜率 | 人工 | 新建 | ≥60% |
| G3/G4 | SimilarityGuard 告警率 / 拦截后二次命中率 | 自动 | 新建 | 告警 <1%；二次命中≈0 |
| G4 | 盲测原作/续写分辨正确率 | 人工 | 新建（50% 为随机线） | ≤60% |
| G4 | 风格距离落入原作自身波动区间的章占比 | 自动 | 每部作品摄入时测定自身区间 | ≥80% |
| G4 | 状态库矛盾 issue 率 | 自动+抽检 | 新建 | <5% |
| G4 | 创意自由度评分（编辑盲评） | 人工 | 新建 | ≥7/10 |
| G4 | 必收伏笔进入规划视野比例 | 自动 | 新建 | 100% |

### 3.3 三类专项评估方法说明

**"更少 AI 味 / 更吸引人"（G2）**
- 主证据是**人工盲测**，自动指标只做回归看护——slop 分数下降但盲测无感=失败；
- 盲测题目用行为化问法（"你愿意读下一章吗""哪段像真人写的"），不用抽象打分；
- 可引入张力类自动指标（如"预测下文成功率"型指标，越难预测张力越高，见 05§3 Spoiler Alert）作为探索性补充。

**"续写像原作者"（G4）**
- 三角验证：读者盲辨（主指标）+ 自动风格距离（统计特征必做；作者嵌入模型可选引入）+ 编辑维度评分（文风/对话/节奏分项）；
- 关键设计：以**原作各章自身波动区间**为判定基线，而非绝对阈值——作者本人也有波动，避免把正常弹性误判为失真；
- 测试集建议：3-5 部已完结作品，隐去真实结尾若干章 → 系统续写 → 与真实后文对照盲测（唯一有 ground truth 的评法）。

**"技法真的被吸收"（G3）**
- 对照实验：同一大纲同一风格，仅切换技法卡开/关，各生成 N 章 → 对应维度（如章末钩子有效性）盲测对比；
- 防伪指标：技法卡启用后 SimilarityGuard 告警率不得上升（吸收的是结构而非表达的证据）。

## 4. 风险登记与缓解

| 风险 | 概率/影响 | 缓解 |
|------|-----------|------|
| 模型锁定不解除，方案 B/C 悬空 | 中/中 | 各目标方案 A 均按 M0 设计为完全可行；解锁按 00§6.2 三步走，每步独立有收益 |
| 多候选成本引发用户抵触 | 中/中 | 成本参数全部用户可控 + 预估报价；fast 模式永远保留 |
| 提示词层反 AI 味失效（约束自身模板化） | 中/高 | 负面清单与人味要素采样轮换（02§4.3/4.4）+ 遥测回归看护 + 方案 C 兜底 |
| 摄入/拆解质量不稳（长篇标注漂移） | 中/高 | 逐章标注带证据引用强制；抽样人工校验（G3-4 指标）；用户校对关键资产（04§4.1） |
| 版权环境变化（尤其输出端判例收紧） | 低/高 | SimilarityGuard 全线强制；来源确认与用途声明落库；05§4.3 条目定期复核；商业化前法务评审 |
| LLM 评审不可靠导致指标失真 | 中/中 | 评审异源 + 人工盲测为主证据 + 评审提示词强制证据引用 |
| 长任务体验差（摄入数十分钟） | 高/低 | AsyncJobService 进度可视 + 分阶段可用（指纹先出→状态库后出） |
| 范围蔓延（四目标并行铺开） | 中/高 | 严格按 §1 顺序；每阶段决策门；本提案明确"仅方案不排期"，立项时逐目标单独立项 |
