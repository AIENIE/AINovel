## 2. 四目标的关系

```
            G2 生成端质量（地基：节拍规划 + 负面约束 + 多候选竞技场）
           ↗            ↖
G1 轻松创作              G3 参考作品拆解 ──共享摄入管线──┐
（向导正文步走 G2 管线）  （技法卡注入 G2 规划阶段）      │
                                                        ▼
                        G4 停更续写（总集成：G2 起草 + G3 摄入 + 指纹 + 契约 + 保真闭环）
```

推荐推进顺序 **G2 → G1 →（G3+G4 摄入共建）→ G4 完全体**，论证见 06 号文档 §1。

## 3. 核心设计立场（贯穿全部提案）

1. **复用优先**：检测端（quality/ 模块、质量模块说明和已落地模式库）已成熟，一律引用不重建；v3 把检测知识反向输入生成端。
2. **AI 味的主战场在情节层与规划层**（事件传送带、无代价爽点、题材同质），不在词句层——见 02 号文档的三层框架。
3. **"像/对/活"的张力显式化**：续写目标中保真（像+对）与创造（活）的矛盾用"自由度契约 + 保真监控闭环"产品化，而非听天由命——见 04 号文档。
4. **技法而非文本**：对他人作品只吸收思想层（结构/节奏/弧线，法律上不受著作权保护），表达层设相似度防线强制拦截——见 03 号文档 §1.2 与 05 号文档 §4.3。
5. **可度量**：每个目标绑定自动指标 + 人工盲测双轨，基线取自现有遥测表——见 06 号文档 §3。

## 4. 已确认的决策边界

- 方案允许包含多模型路由与微调/LoRA 轨道（M1/M2 档），但每个目标的方案 A 必须 M0（现状模型）可行；
- 参考作品/停更原作允许全文私有导入（用户私人分析与创作用途），平台侧保留来源确认、用途声明与输出相似度防线；
- 除 G2 Step 1 与 G1-A P0 外，本目录内容仍为提案；统一排期与门槛以 `PENDING-PHASES.md` 为准。
