# 正文写作

## 基本操作

1. 在“小说创作”选择故事、大纲和稿件；没有稿件时先新建。
2. 从左侧大纲树打开一个场景。
3. 在编辑器中直接写作，或点击“生成本场景”。
4. 使用“保存”保留手动修改；未保存场景会显示标记。

生成正文会遵守当前场景、大纲、角色、前文、世界观和可用素材上下文。服务端要求每节正文处于目标字数区间，并会记录文本质量与剧情质量结果。

## 快速与精雕

桌面版编辑器的“快速/精雕”开关决定下一次“生成本场景”的起草方式：

- “快速”：默认模式，使用标准场景起草规则。
- “精雕”：在标准规则上增加反套路约束和叙事目标，适合需要更多具体感、关系变化、代价或节奏缓冲的场景。

精雕模式仍是单次候选生成，不提供节拍方案选择、多候选对比或自动选稿。模式不会在页面刷新后保留；每次进入写作页默认回到“快速”。当前移动版编辑器不提供场景生成和模式切换控制。

## 参与 G2 盲测

当管理员开放投稿活动时，桌面编辑器会显示“参与盲测”。选择活动后，系统会为当前场景生成一份快速文本和一份精雕文本，用于匿名对照；它们不会写回正在编辑的正文、版本或剧情质量记录。每个场景在同一活动中只能提交一次。

活动详情和评审规则见 [G2 盲测](10-G2盲测.md)。

## 编辑器与侧栏

- 使用编辑器工具栏进行基础富文本编辑和文本润色。
- 右侧可查看 Copilot、上下文、剧情、版本、导出、统计和目标等面板。
- 质量状态徽标反映最近一次场景质量记录；它是修改建议，不会替代作者判断。

## 角色变化与素材建议

生成或保存后的场景可用于角色变化分析和素材提示。若建议与当前故事设定冲突，应优先采用故事和大纲中的明确设定。
