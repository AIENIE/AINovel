# 质量、版本与导出

## 质量分析

工作台的“质量分析”和稿件右侧“剧情”面板提供三类检查：

- 文本 Slop 诊断：定位模板化表达、风格贴合、承接和叙事节奏风险。
- 长篇 drift 巡检：对足够长的稿件分窗口检查后段模板化、角色漂移和伏笔遗忘。
- 剧情质量诊断：检查目标冲突、因果、角色能动性、风险收益、伏笔与读者期待。

分析结果提供风险分、证据和最小修改建议。它不会输出“AI 率”或作者归因。剧情分析可以生成候选修订，但只有点击“采纳”后才会写回正文；采纳前会再次检查正文是否已变化。

## 版本与分支

在稿件的“版本”面板中可以：

- 创建手动检查点。
- 查看版本、比较差异和回滚到历史版本。
- 创建、切换、更新、合并或废弃稿件分支。
- 配置自动保存的间隔和最多保留版本数。

在大幅生成、接受候选修订或结构性重写前，建议先创建检查点。

## 导出

在“导出”面板选择格式和模板后创建导出任务。任务完成后下载文件。导出任务保留元数据，下载内容按当前稿件生成，因此应在导出前确认已保存所需场景。
